## 1. 각 게이트 상세 증거

### GATE 1: `npm run lint` — pre-existing 실패

**HEAD (T-001 stash 된 상태) 실행**:
```
$ git stash && npm run lint
> claude-agent-manager@0.1.0 lint
> next lint
Invalid project directory provided, no such directory: /Users/min/Documents/claude-agent/lint
```

Next.js 16 에서 `next lint` 가 deprecated 되어 package.json 의 lint 스크립트가 더 이상 정상 동작 안 함. **`next lint <dir>` 인자 해석 버그**로 존재하지 않는 `/lint` 경로를 찾으려 시도. pre-existing.

**수정 방안 (T-001 범위 밖)**: package.json 의 `lint` 스크립트를 ESLint 직접 호출 (`eslint .` 또는 `eslint src`) 또는 Biome 같은 대체 도구로 이관.

### GATE 2: `npm run build` — pre-existing 실패

**실행 결과**:
```
⨯ useSearchParams() should be wrapped in a suspense boundary at page "/templates".
Error occurred prerendering page "/templates".
Export encountered an error on /templates/page: /templates, exiting the build.
```

원인: `/src/app/templates/page.tsx` 최상단 `useSearchParams()` 훅이 `<Suspense>` 경계 없이 쓰임 — Next.js 16 엄격 SSG 요구. **이 코드는 T-001 diff 이전부터 존재** (line 87 `const searchParams = useSearchParams()`, 내 diff 건드리지 않음).

**수정 방안 (T-001 범위 밖)**: `page.tsx` 를 `<Suspense>` 래퍼로 감싸고, 실제 컴포넌트를 내부로 분리. 또는 `export const dynamic = "force-dynamic"` 설정.

`test` 키 없음. 테스트 프레임워크(Vitest/Jest/Playwright 등) 미설치. pre-existing.

### GATE 4: `e2e-before-commit.sh` — hook 자체 버그

**trace (bash -x) 마지막 2줄**:
```
++ ls -t '.claude/pipeline/alerts/ALERT-*.md'
++ head -1
+ ALERT_FILE=
```

line 36 에서 `ls -t .claude/pipeline/alerts/ALERT-*.md 2>/dev/null | head -1` — `alerts/` 에 ALERT-*.md 없으면 `ls` 가 exit code 1~2 반환. `set -o pipefail` 때문에 파이프라인 전체 실패 → `set -e` 로 스크립트 중단. stdout 빈 채 exit 1.

**HEAD 에서도 동일 실패** (stash 상태에서 확인):
```
$ git stash && echo '{...}' | bash .claude/hooks/e2e-before-commit.sh > /tmp/h1.out 2> /tmp/h1.err
HOOK_EXIT=1
# /tmp/h1.out: 0 byte
# /tmp/h1.err: 0 byte
```

**수정 방안 (T-001 범위 밖, 그리고 `.claude/hooks/**` 는 Implementer deny)**:
line 36 을 다음으로 변경 필요:
```bash
ALERT_FILE=$(ls -t .claude/pipeline/alerts/ALERT-*.md 2>/dev/null | head -1 || true)
```
또는 shopt -s nullglob 사용. 이건 별도 SPEC + Reviewer/Supervisor 권한.

### 브라우저 실측 증거 수집 제안 (Reviewer/사용자 영역)

**권장 수동 절차 (10초)**:

4. Apply 버튼 클릭
5. Network 탭에서 `batch-apply` 요청 클릭 → Payload 확인 (기대: `{templateIds, scope:"project", projectPath:"/tmp/test-claude-project", mode:"merge"}`)
6. Response 200 + `{success:true, applied:1}` 확인
7. 터미널: `cat /tmp/test-claude-project/.claude/settings.json` 으로 변경 확인
