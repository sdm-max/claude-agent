## 0. 결론 (TL;DR)

**Apply 버튼은 2개 진입점**. 두 진입점 모두:

| 필드 | 전송 여부 | 잘못된 점 |
|------|---------|----------|
| `scope` | ✅ 있음 | **기본값 `"global"`** (applyScope / dialogScope 모두). 사용자 의도와 불일치 가능 |
| `projectPath` | ✅ 조건부 (needsProject 이면 포함, 아니면 생략) | 없음 — 백엔드가 global/user 에선 projectPath 불필요로 허용 |
| `mode` | ✅ 고정 `"merge"` | 없음 |

**누락 필드 없음**. `templateIds` / URL path 형태도 백엔드 스펙과 일치.

**잘못된 유일한 점**: `scope` 기본값 `"global"` 이라 사용자가 토글 안 바꾸면 apply 대상이 `~/.claude/managed-settings.json` 이 됨 → 사용자가 기대한 프로젝트 settings.local.json 은 미변경 → "apply 안 됨" 오해.

## 1. 진입점 #1 — Apply Bar 의 "Apply N templates" 버튼

### 1-1. onClick 연결
**파일**: `src/app/templates/page.tsx`  
**라인 890~895**:
```tsx
<Button
  onClick={batchApply}
  disabled={
    applying || (needsProject && !selectedProjectPath)
  }
  size="sm"
>
```

### 1-2. `batchApply` 함수 전체
**라인 322~367**:
```ts
const batchApply = async () => {
  if (selected.size === 0) return;
  const needsProject = applyScope === "project" || applyScope === "local";
  if (needsProject && !selectedProjectPath) return;

  if (
    conflictReport?.hasCritical ||
    (conflictReport?.orderDependencies?.length ?? 0) > 0
  ) {
    const orderMsg = conflictReport?.orderSummary
      ? `\n\n순서 의존: ${conflictReport.orderSummary}`
      : "";
    const ok = confirm(
      `위험: ${conflictReport?.summary || ""}${orderMsg}\n\n계속 적용하시겠습니까?`,
    );
    if (!ok) return;
  }
  setApplying(true);
  setApplyResult(null);
  try {
    const res = await fetch("/api/templates/batch-apply", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        templateIds: [...selected],
        scope: applyScope,
        projectPath: needsProject ? selectedProjectPath : undefined,
        mode: "merge",
      }),
    });
    const data = await res.json();
    if (res.ok) {
      const target = needsProject
        ? `${applyScope} (${selectedProjectPath.split("/").pop()})`
        : applyScope;
      setApplyResult(`${data.applied} templates applied to ${target}`);
      loadAppliedMap();
    } else {
      setApplyResult(`Error: ${data.error}`);
    }
  } catch {
    setApplyResult("Failed to apply");
  } finally {
    setApplying(false);
  }
};
```

### 1-3. 실제 전송되는 body (JSON.stringify 이후)
```json
{
  "templateIds": ["<선택된 카드 id들>"],
  "scope": "<applyScope state 값>",
  "projectPath": "<needsProject 일 때만. 아니면 생략됨>",
  "mode": "merge"
}
```

### 1-4. state 초기값
**라인 164**:
```ts
const [applyScope, setApplyScope] = useState("global");
```

**라인 166**:
```ts
const [selectedProjectPath, setSelectedProjectPath] = useState("");
```
(projects fetch 완료 후 첫 프로젝트 path 로 덮어씀)

### 1-5. needsProject 판별
**라인 324**:
```ts
const needsProject = applyScope === "project" || applyScope === "local";
```

→ scope 가 `"global"` 또는 `"user"` 일 때는 **projectPath 가 body 에서 생략**됨 (JSON.stringify 가 undefined 필드 제거).

## 3. 필드별 문제 여부

| 필드 | 전송 | 문제 | 상세 |
|------|------|------|------|
| `scope` | ✅ 항상 포함 | **🔴 기본값이 `"global"`** | 사용자가 토글 안 바꾸면 managed-settings.json 에 기록. 테스트 프로젝트 settings.local.json 은 변화 없음 → "apply 안 됨" 오해 |
| `projectPath` | ⚠️ 조건부 | 🟢 없음 | needsProject 때만 포함 (백엔드 스펙 일치). global/user 에서 없는 게 정상 |
| `mode` | ✅ 항상 `"merge"` | 🟢 없음 | 고정값. 추후 "replace" 옵션 원하면 UI 추가 (범위 밖) |
| `templateIds` (Bar) | ✅ 선택된 ID 배열 | 🟢 없음 | 복수 카드 batch 용 |
| URL path id (Dialog) | ✅ `/api/templates/${detail.id}/apply` | 🟢 없음 | single apply 백엔드 스펙 일치 |

## 4. scope 기본값의 실질 영향 (원인 A 재확인)

### 시나리오 A-1 (사용자가 scope 안 바꿈)
1. `/templates` 페이지 진입 → `applyScope = "global"`
2. 카드 선택 → Apply Bar 에 Apply 버튼 활성 (needsProject=false 므로 projectPath 선택 안 해도 disabled 안 됨)
3. 클릭 → body = `{templateIds, scope: "global", mode: "merge"}` 전송
4. 백엔드 `/api/templates/batch-apply` → `resolveSettingsPath("global", ...)` → `~/.claude/managed-settings.json` 에 기록
5. 사용자는 Test Project 의 `.claude/settings.local.json` 을 확인 → 변화 없음 → "apply 안 됨"

### 시나리오 A-2 (사용자가 scope 를 local 로 바꿈)
1. scope 토글 클릭 → `applyScope = "local"`
2. `needsProject = true` → 프로젝트 선택 dropdown 노출
3. 프로젝트 선택 필수 (없으면 버튼 disabled)
4. Apply → body = `{templateIds, scope: "local", projectPath: "/tmp/test-claude-project", mode: "merge"}`
5. 백엔드 → `/tmp/test-claude-project/.claude/settings.local.json` 에 기록 ✓
6. 성공. 이게 내가 사전 작업 curl 로 확인한 경로.

→ 시나리오 A-1 이 "apply 안 됨" 증상 재현. 가설 A 확정.

## 5. 결론 (Reviewer 요청 답)

- **누락 없음**: scope/projectPath/mode 3 필드가 JSON body 또는 URL path 에 모두 포함됨
- **잘못된 점 (1개)**: `scope` 기본값 `"global"` — 대부분 사용자 의도는 "이 프로젝트에 apply" 이므로 기본이 global 이면 오해 발생
- **수정 타겟**: `src/app/templates/page.tsx:164, 179` (applyScope, dialogScope 초기값)
