## Scope
- **포함**: `src/app/templates/page.tsx` 하나만 수정
  - 기존 export default function TemplatesPage() 를 내부 컴포넌트(예: TemplatesPageInner)로 이름 변경
  - 새 default export 는 `<Suspense fallback={<div>Loading...</div>}><TemplatesPageInner /></Suspense>` 래퍼
- **제외**: 다른 페이지의 `useSearchParams()` 사용 조사/수정, 라이브러리 업그레이드, dynamic export config 변경

## 가정
- Next.js 16 의 SSG 엄격 모드는 `useSearchParams()` 가 Suspense 경계 없이 쓰일 때 prerender 실패
- Suspense fallback 으로 단순 로딩 텍스트만 표시해도 UX 허용
- **가장 위험한 가정**: Suspense 래핑이 기존 state/useEffect 타이밍에 영향 없음 (React 18+ 표준 동작 가정)

## Edge Case
1. `?category=custom` 같은 query 진입 시 fallback 순간 표시 → 문제없음 (이전에도 hydration 전 빈 상태)
2. 중첩 `useSearchParams` 다른 hook 없음 (해당 파일에서 최상단 1회만)
3. Client-side navigation 시 Suspense 안에서 재평가 — 동일 동작

## 성공 판정 명령

npm run build 2>&1 | grep -E "useSearchParams|prerender-error|Export encountered" && echo "FAIL" || echo "PASS"
