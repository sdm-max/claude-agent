## Scope

- **제외**: 다른 Guard 로직, 전체 hook 리팩터, 다른 `ls | head` 사용처

## 가정
- `.claude/hooks/**` Edit/Write 는 원래 Implementer deny → **이 T-002 에 한해 권한 예외** (사용자 명시)
- 수정 후 hook 실행은 기존 Guard 0d (ALERT 확인) / 0c (outbox) / TSC / dev ping / e2e-scenarios 흐름 그대로 유지
- **가장 위험한 가정**: `|| true` 는 exit code 를 덮어쓰므로 pipefail 의도한 실제 에러까지 숨길 수 있음 — 하지만 이 특정 line 에서는 "매치 없음" 이 정상 케이스라 허용 가능

## Edge Case
1. ALERT 파일 여러 개 있을 때 → 기존 `head -1` 동작 유지 (가장 최신)
2. ALERT 디렉토리 자체 없음 → `2>/dev/null` 로 stderr 억제, `|| true` 로 exit 0
3. glob 매치 0 개 → 본 수정 대상 (pipefail 우회)
