## 2. 원인 후보 (우선순위)

### A. 기본 `applyScope = "global"` — 사용자 의도 불일치 (최고 가능성) 🔴

- `const [applyScope, setApplyScope] = useState("global")` — `src/app/templates/page.tsx:164`
- 기본값 **`global`** → `needsProject = false` → projectPath 안 보냄 → 백엔드가 `~/.claude/managed-settings.json` 에 기록.
- 사용자는 보통 **특정 프로젝트(Test Project 등) 에 apply** 하길 원함.
- scope 토글 바꾸지 않고 Apply 버튼 클릭 → managed-settings.json 에 쓰여지는데, 사용자는 **프로젝트의 `.claude/settings.local.json` 확인** → "apply 안 됨" 오해.
- **증거**: 방금 curl 로 scope=local 로 호출 시 `/tmp/test-claude-project/.claude/settings.local.json` 에 정상 기록 확인 (사전 작업에서).

### B. `needsProject && !selectedProjectPath` 시 early return — 피드백 없음 🟡

- `src/app/templates/page.tsx:325`: `if (needsProject && !selectedProjectPath) return;`
- 프로젝트 scope 로 바꿨지만 프로젝트 선택 안 한 경우 — 버튼이 `disabled` 되어야 하지만 edge case 있을 수 있음 (projects 로딩 전 클릭 등).
- 버튼 disabled 로 막고 있으나 (line 893), **disabled 도 silent** — 왜 안 되는지 안 보임.

### C. catch 블록 silent 🟡

- `src/app/templates/page.tsx:362-364, 436-438`:
  ```ts
  } catch {
    setApplyResult("Failed to apply");
  }
  ```
- `console.error` 없음. 네트워크 에러/500 에러 시 DevTools 에 흔적 안 남아 원인 파악 어려움.

### D. 인라인 결과 메시지 가시성 낮음 🟡

- `applyResult` / `dialogApplyResult` 는 `<span className="text-xs ...">` 로 렌더. Apply Bar / Dialog 안 작은 텍스트.
- 성공/실패 눈에 잘 안 들어옴. 사용자가 결과 못 봤을 수 있음.

## 3. 제안 수정 (TASKS 분해 제안)

### T-001.1 — 기본 scope 안전한 값으로 (A 해결)
**파일**: `src/app/templates/page.tsx:164`
- 변경: `useState("global")` → 첫 프로젝트가 등록돼 있으면 `"project"`, 없으면 `"user"` 로 기본값.
- 이점: 사용자가 scope 선택 안 해도 "현재 프로젝트에 적용" 의도와 일치.
- 리스크: 기존 UX 변화. 사용자가 global 의도라면 오히려 혼란 가능 → 대안: scope selector 에 색상/아이콘/경고 표시.

### T-001.2 — 프로젝트 미선택 시 명확한 피드백 (B 해결)
**파일**: `src/app/templates/page.tsx:322-325, 384-387`
- 변경: early return 대신 `setApplyResult("프로젝트를 선택하세요")` 혹은 버튼 hover tooltip.

### T-001.3 — catch 에 `console.error` + 상세 에러 (C 해결)
**파일**: `src/app/templates/page.tsx:362-364, 436-438`
- 변경: `catch (e) { console.error("Apply failed:", e); setApplyResult(\`Failed: ${e.message ?? "network"}\`); }`

### T-001.4 — Apply 결과 피드백 강화 (D 해결)
**파일**: `src/app/templates/page.tsx:357-358, 408`
- 변경: 성공 시 실제 기록된 파일 경로 또는 scope 이름을 **강조된** 배지/토스트로 표시.
- 범위: 큰 변경일 수 있어 별도 task 로 분리.
